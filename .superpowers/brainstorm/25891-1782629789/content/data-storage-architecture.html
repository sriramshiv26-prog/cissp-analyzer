Where Should Historical Exam Data Live?

This is the foundation for scaling to 10+ exams per student

A

File-Based Storage

Store: JSON/Excel files in project directory

✓ Simple, no DB setup

✓ Version control friendly (git)

✗ Poor query performance (10+ exams = slow)

✗ Concurrent access issues (multiple students)

✗ Hard to aggregate class-level analytics

B

PostgreSQL (Relational)

Store: Exam history in existing Cyber Sierra DB

✓ Fast queries (indexed on student + domain)

✓ ACID guarantees (data integrity)

✓ Easy aggregations (class analytics)

✗ Requires schema design + migrations

✗ Tighter coupling to Cyber Sierra

C

Hybrid (Excel + Cache)

Store: Excel files + in-memory cache during analysis

✓ Standalone (works without Cyber Sierra)

✓ User-friendly (drag-drop Excel files)

✓ Still version-controllable

✗ Cache expires on restart

✗ Still has file-based query limits

D

MongoDB (Document)

Store: Exam history in Second Brain DB

✓ Flexible schema (variable question counts)

✓ Fast document queries

✓ JSON-native (matches our data structure)

✗ Requires Second Brain integration

✗ Extra deployment dependency
