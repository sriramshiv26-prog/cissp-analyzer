🔄 Data Flow (End-to-End)

Step 1: Input
User: "Enter student name?" → "Sri"
Tool scans: students/Sri/ folder → finds 2 previous exams
Loads: exam-1_performance.json, exam-2_performance.json

Step 2: Parse Current Exam
Reads: Mock3_Jul1_Sri.xlsx
Extracts: 125 questions, right/wrong per student
Calculates: 5-dimensional performance (domain, topic, difficulty, type, trick)

Step 3: Merge with History
Compares: Current exam (exam-3) vs exam-1, exam-2
Detects: Trends, improvements, regressions per dimension

Step 4: Generate Recommendations
Algorithm (Momentum-Based):
  For each domain:
    momentum = (current - previous) if history exists else 0
    weakness = 100 - current
    priority = weakness + (momentum × 2)
  Rank domains by priority
  Drill down to subtopics + patterns

Step 5: Generate Reports
Creates: Sri_progress_report.xlsx (8 sheets)
Stores: exam-3_performance.json for next run
