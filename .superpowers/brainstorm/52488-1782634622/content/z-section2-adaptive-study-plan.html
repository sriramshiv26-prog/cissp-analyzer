🧠 Momentum-Based Recommendation Algorithm

Weighs weakness + improvement trajectory to prioritize study focus

Algorithm: Calculate Priority Score Per Domain
FOR each domain:
  current_accuracy = exam-3 accuracy %
  previous_accuracy = exam-2 accuracy % (if exists)
  
  momentum = (current - previous) if history else 0
  weakness = 100 - current
  priority_score = weakness + (momentum × 2)
  
SORT domains by priority_score descending
RECOMMEND top 2-3 domains for focused study

📊 Worked Example (Real Student Data)

Software Development Security 55% 58% 42 +3 48 🥈 #2
Security Architecture 72% 75% 25 +3 31 🥉 #5

💡 Key insight: Cryptography dropped 80% → 78% but still ranks #8 because weakness (22) is low. Identity & Access ranks #1 because it has both high weakness (50) AND positive momentum (+5).
