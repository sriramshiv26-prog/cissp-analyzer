================================================================================
KEY FORMATS EXPLAINED
================================================================================

1. ANSWER KEY JSON
   ├─ File: exams/exam_[N]_[DESC]_answer_key.json
   ├─ Contains: 125 questions with answers and explanations
   ├─ Format: {"1": {"letter": "A", "text": "..."}, ...}
   ├─ Must have: All 125 questions, valid JSON
   └─ See: TEMPLATE_answer_key.json

2. STUDENT ANSWERS EXCEL
   ├─ File: answers/batch_[NAME]/exam_[N]_batch_[NAME].xlsx
   ├─ Contains: Student responses in columns, Q1-Q125 in rows
   ├─ Formats: A, B, C, D, or 1-A,2-B,3-C,4-D (auto-normalized)
   ├─ Must have: 125 rows, unique student names, .xlsx format
   └─ See: TEMPLATE_student_answers.md

3. DIRECTORY STRUCTURE
   ├─ Layout: exams/ + answers/ + students/ + outputs/
   ├─ Batches: Use subfolders for cohorts (batch_dec25, batch_july26)
   ├─ Naming: Lowercase with underscores, no spaces
   ├─ Organization: Batch vs Standalone separation
   └─ See: TEMPLATE_directory_structure.md

================================================================================
ANSWER FORMAT VARIATIONS (ALL SUPPORTED)
================================================================================

The tool supports 6 different answer formats - all auto-normalized:

Format 1: Single Letter
Input:  A
Output: A (normalized)

Format 2: Letter with Dash
Input:  1-A
Output: 1-A (normalized)

Format 3: Multi-Part with Dashes
Input:  1-A,2-B,3-C,4-D
Output: 1-A,2-B,3-C,4-D (normalized)

Format 4: Multi-Part No Separators
Input:  1A2B3C4D
Output: 1-A,2-B,3-C,4-D (normalized)

Format 5: Positional (Auto-Numbered)
Input:  A,B,C,D
Output: 1-A,2-B,3-C,4-D (normalized)

Format 6: Lowercase/Spaces
Input:  a, b, c, d
Output: 1-A,2-B,3-C,4-D (normalized)

This flexibility means users can enter answers in whatever format they're
comfortable with - the tool automatically converts to standard format!

================================================================================
VALIDATION BEFORE RUNNING ANALYSIS
================================================================================

Use this checklist from FORMATS_AND_TEMPLATES_GUIDE.md:

ANSWER KEY JSON ✓
- Valid JSON syntax (test with jsonlint.com)
- Exactly 125 questions (keys 1-125)
- Each has "letter" (A/B/C/D) and "text"
- All answers present (no null values)

STUDENT ANSWERS EXCEL ✓
- File is .xlsx format (not .xls or .csv)
- Exactly 125 rows (Q1-Q125)
- Student names in header (no blanks)
- Answers are A/B/C/D or multi-part format
- No merged cells or special formatting

DIRECTORY STRUCTURE ✓
- /exams/ contains PDF + answer key JSON
- /answers/ organized by batch or standalone
- /students/ exists (auto-populated)
- /outputs/ exists (can be empty)
- All lowercase names with underscores

NAMING CONVENTIONS ✓
- Answer key: exam_[N]_[DESC]_answer_key.json
- PDF: exam_[N]_[DESC].pdf
- Excel: exam_[N]_batch_[NAME].xlsx
- All lowercase, underscores only

QUICK REFERENCE: BEFORE YOU RUN ANALYSIS
================================================================================

Use this 60-second validation:

1. Answer Key JSON
   python3 -c "import json; json.load(open('exams/exam_1_answer_key.json'));
               print('✓ Valid JSON')"
   ✓ Check output says "Valid JSON"

2. Excel File
   - Open in Excel/Google Sheets
   - Count rows: should be ~126 (header + 125 questions)
   - Spot check: Row 2 = Q1, Row 3 = Q2, Row 126 = Q125
   - Verify student name in Column header

3. Directory Structure
   ls -R exams/ answers/ students/ outputs/
   ✓ Check all folders exist and files are in right places

4. File Names
   - exams/exam_1_*_answer_key.json ✓
   - answers/batch_*/exam_1_*.xlsx ✓
   - No spaces in any name ✓

5. Run Analysis
   python3 analyze.py

If anything fails, check FORMATS_AND_TEMPLATES_GUIDE.md "Troubleshooting"
section!
